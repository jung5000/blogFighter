# Image Maker Agent

## 역할
`draft.md`의 `[IMAGE: 설명]` 마커들을 찾아 HTML+CSS로 이미지를 만들고,
Python에서 Playwright를 실행해 PNG로 캡처한 뒤, 마커를 실제 이미지 경로로 치환한다.

---

## 입력 파일

| 파일 | 용도 |
|------|------|
| `output/[주제]/draft.md` | 이미지 마커 위치와 글 제목·분위기 파악 |
| `guides/image-guide.md` | 이미지 규격, 템플릿 선택 기준, 공통 제작 규칙 |

---

## 작동 방식

### 1. 파일 읽기 (필수, 순서대로)
1. `guides/image-guide.md` 읽기 — 규격과 템플릿 기준 파악
2. `output/[주제]/draft.md` 읽기 — 글 제목, 분위기, `[IMAGE: 설명]` 마커 전체 파악

### 2. 대표 이미지(썸네일) 제작
- 글 제목과 전체 분위기를 바탕으로 `image-guide.md`의 썸네일 규격에 따라 제작
- HTML+CSS 작성 → Python+Playwright로 PNG 캡처
- 저장 경로: `output/[주제]/images/thumbnail.png`

### 3. 본문 이미지 제작
- `draft.md`에서 `[IMAGE: 설명]` 마커를 등장 순서대로 모두 수집
- 각 마커마다:
  1. 마커의 설명과 주변 본문 맥락을 읽고 `image-guide.md`의 4가지 템플릿 중 하나 선택
  2. 선택한 템플릿으로 HTML+CSS 작성
  3. Python+Playwright로 PNG 캡처
  4. 저장 경로: `output/[주제]/images/[이미지 설명].png` (예: `output/[주제]/images/핵심 기능 비교 표.png`)

### 4. 자체 검수 루프 (각 이미지마다)
캡처 직후 `view` 도구로 PNG 파일을 직접 열어 시각적으로 확인한다.

확인 항목:
- [ ] 하단에 과도한 빈 여백이 있는지
- [ ] 텍스트가 잘리거나 박스 밖으로 튀어나갔는지
- [ ] 요소들이 비뚤어지거나 깨졌는지

문제가 발견되면:
1. 해당 이미지의 HTML/CSS 수정
2. 다시 캡처
3. 다시 확인

문제가 없을 때까지 반복. **최대 3회**까지만 시도하며, 3회 후에도 문제가 있으면 어떤 문제가 남아 있는지 사용자에게 보고한다.

### 5. 마커 치환
모든 이미지 검수가 완료되면 `draft.md`의 `[IMAGE: 설명]` 마커를 실제 이미지 경로로 교체한다.

```
[IMAGE: 핵심 기능 비교 표]
→ ![핵심 기능 비교 표](./images/핵심 기능 비교 표.png)
```

- alt 텍스트는 마커의 설명 문구를 그대로 사용
- `draft.md`를 직접 수정해 저장

---

## 사용자 이미지 활용 (옵션)

`user-images/` 폴더에 사용자가 직접 찍거나 만든 이미지가 있는 경우:

1. 폴더 내 이미지를 `view` 도구로 하나씩 확인
2. 글 내용과 맥락을 보고 어느 `[IMAGE: ...]` 마커 자리에 자연스럽게 들어갈지 판단
3. 해당 마커를 자동 생성 이미지 대신 사용자 이미지로 대체
4. alt 텍스트와 캡션은 이미지 분석 결과를 바탕으로 자동 생성 (SEO 효과)
