# Image Guide

이미지 메이커 에이전트가 이미지를 만들 때 따르는 규격과 템플릿 가이드.

---

## 대표 이미지 (썸네일) 규격

| 항목 | 규격 |
|------|------|
| 크기 | 가로 1080px × 세로 1080px (1:1 정사각형) |
| 배경 | 어두운 블루-퍼플 그라데이션 (`#1a1a2e` → `#16213e`) |
| 메인 텍스트 | 글 제목 — 큰 글씨, 흰색 |
| 보조 텍스트 | 카테고리 또는 부제 — 작은 글씨, 연한 회색 |
| 정렬 | 모든 텍스트 가로·세로 중앙 정렬 |
| 폰트 | Pretendard 또는 시스템 기본 한글 폰트 |
| 액센트 요소 | 우측 하단 또는 좌상단에 작은 도형, 코드 심볼, 또는 단순 아이콘 |

---

## 본문 삽입 이미지 — 4가지 템플릿

글 내용과 [IMAGE: 설명] 마커의 설명을 보고 아래 네 가지 중 가장 적합한 템플릿을 선택한다.

### 1. 비교 표
**언제**: 제품, 서비스, 방법을 나란히 비교할 때  
**구성**: 헤더 행 + 항목 행, 열마다 구분선, 강조 열 표시 가능

### 2. 단계별 다이어그램
**언제**: 절차나 순서를 시각적으로 보여줄 때  
**구성**: 번호 붙은 단계 박스들을 화살표로 연결 (가로 또는 세로)

### 3. 핵심 포인트 카드
**언제**: 3~5개 요점을 한눈에 정리할 때  
**구성**: 각 포인트를 독립 카드로 배치, 아이콘 또는 번호 + 짧은 텍스트

### 4. 인용/강조 박스
**언제**: 중요한 한 마디나 핵심 문장을 강조할 때  
**구성**: 큰 따옴표 또는 좌측 강조선 + 인용 텍스트 + 출처(선택)

## 공통 제작 규칙

- **제작 방식**: HTML + CSS로 마크업 작성 → Python에서 Playwright로 PNG 캡처
- **배경 톤**: AI/테크 분위기에 맞는 어두운 톤 유지 (딥 네이비, 다크 퍼플 계열)
- **텍스트 원칙**: 최대한 짧게. 이미지는 보조 수단이며 텍스트의 반복이 아님
- **코드 스타일 활용**: 모노스페이스 폰트와 어두운 배경을 디자인 요소로 활용 가능
- **저장 위치**:
  - 대표 이미지: `output/[주제]/images/thumbnail.png`
  - 본문 이미지: `output/[주제]/images/[이미지 설명].png` (마커의 '이미지 설명'을 파일명으로 사용)
